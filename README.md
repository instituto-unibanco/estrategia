# Estratégia de Desenvolvimento de Software do Instituto Unibanco

O desenvolvimento de software no Instituto Unibanco segue diretrizes que nos permitem contribuir de forma eficiente para os principais desafios da Educação.

Este manifesto declara as principais escolhas que definem nossa estratégia.

## 1. Todo software deve ser livre

Todo software desenvolvido deve ser livre para uso, modificação e redistribuição. Isto permite que:

- Barreiras burocráticas sejam minimizadas para facilitar a adoção das nossas ferramentas.
- Trabalhemos como parte de uma comunidade maior, que aprimore, estenda e contribua com nossos projetos.

## 2. Todo software deve servir à Educação

Todo software produzido deve ser uma ferramenta que viabilize a missão do Instituto Unibanco na Educação.

## 3. Criação de novo software deve ser o último recurso

Código é um passivo. É importante mantê-lo mínimo e essencial. Sua existência deve atender a um propósito claro que ainda não seja atendido por nenhum software livre disponível.

Por ordem de preferência, deve-se:

1. Usar software existente.
2. Estender software existente.
3. Divergir de software existente (_Fork_).
4. Criar software próprio.

## 4. Modularidade deve ser priorizada

Deve ser possível utilizar componentes independentes do software principal. As interfaces entre componentes devem ser claras para facilitar a interopeabilidade com outros sistemas.

## 5. Barreiras técnicas devem ser minimizadas

Barreiras técnicas criam desigualdade. Minimizá-las contribue para a democratização dos benefícios que nossas ferramentas podem oferecer à comunidade. Portanto, o processo de deploy e operação de software deve exigir passos mínimos e de baixa complexidade.

## 6. Infraestrutura deve ser uma escolha livre

Acreditamos que plataformas devem ser disponibilizadas para o usuário final em parceria com provedores de infraestrutura.

Deve-se garantir a liberdade de escolha ao operador através do suporte a múltiplas alternativas de deploy da plataforma e de suas dependências.

Em nome da facilidade, escalabilidade e confiabilidade da operação, a plataforma pode depender de soluções proprietárias de infraestrutura. Contudo, deve sempre existir uma alternativa equivalente que torne essas dependências dispensáveis.
